# 02. 요구사항 분석 (Requirements Analysis)

본 문서는 거래소 시스템이 제공해야 할 핵심 기능과 성능적 목표를 정의합니다.

## 1. 기능적 요구사항 (Functional Requirements)
*   **주문 관리 시스템 (OMS):**
    *   지정가(Limit Order): 특정 가격에 도달 시 체결.
    *   시장가(Market Order): 현재 호가에서 즉시 체결.
    *   주문 취소 및 수정 기능 제공.
*   **사용자 자산 관리:**
    *   실시간 잔고 업데이트 (체결 시 즉시 반영).
    *   입출금 처리 시 자산 락(Lock) 및 언락(Unlock) 메커니즘.
*   **시세 및 통계:**
    *   실시간 호가창(Orderbook) 데이터 생성.
    *   타임 프레임별(1m, 5m, 1h, 1d) 캔들 차트(OHLCV) 데이터 생성.
    *   최근 체결 내역(Recent Trades) 제공.

## 2. 비기능적 요구사항 (Non-Functional Requirements)
*   **초저지연성 (Ultra-Low Latency):**
    *   매칭 엔진 내부의 주문 처리 지연 시간을 3ms 이내로 유지.
    *   End-to-End 지연 시간(주문 접수 ~ 체결 알림)을 최소화.
*   **고가용성 (High Availability):**
    *   시스템 장애 시 99.99% 이상의 가동률 보장.
    *   장애 발생 시 10초 이내에 예비 엔진으로 페일오버(Failover).
*   **확장성 (Scalability):**
    *   거래 페어 증가 시 매칭 엔진 인스턴스를 수평적으로 확장 가능한 구조.
    *   동시 접속자 수 10만 명 이상 처리 가능.
*   **데이터 무결성:**
    *   매칭 엔진의 체결 결과와 DB의 자산 정보 간에 일말의 오차도 허용하지 않음.
